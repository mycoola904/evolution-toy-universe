# 02 - Core Principles

## Purpose

The Evolution Toy Universe is intended to explore the emergence of complex behavior from extremely simple rules.

The objective is not to accurately simulate biology, chemistry, or physics. Instead, the project seeks to create a minimal artificial universe in which lifelike behaviors can emerge naturally.

Every design decision should favor simplicity, consistency, and emergence over realism.

---

# Guiding Principles

## 1. Emergence over Prescription

Interesting behavior should arise from interactions between simple rules rather than being explicitly programmed.

Organisms should never be given concepts such as:

- predator
- prey
- hunger
- fear
- intelligence
- reproduction strategy

Instead, these should emerge naturally if the underlying rules allow them.

---

## 2. Simple Rules First

Whenever two possible implementations exist, prefer the simpler one.

Complexity should emerge from interactions—not from complicated rules.

---

## 3. Everything Has a Cost

Every action performed by an organism consumes energy.

Examples include:

- movement
- thinking
- sensing
- reproduction
- memory
- communication

Nothing is free.

---

## 4. Information is Local

Organisms know only what they can directly perceive.

There is:

- no global knowledge
- no perfect information
- no magical sensing

Every decision is made using local observations.

---

## 5. Continuous Evolution

There is no "winning."

The simulation should never converge toward a fixed goal.

Evolution is an ongoing process.

---

## 6. Consistent Physical Laws

The universe should obey the same rules everywhere.

If energy behaves one way in one location, it behaves that way everywhere.

Rules should never change simply to help organisms survive.

---

## 7. Small Building Blocks

Prefer many tiny reusable mechanisms over specialized systems.

For example:

Instead of creating a "Predator" class, create movement, sensing, collision, and energy transfer.

Predation should become an emergent consequence.

---

## 8. Determinism First

The simulation should be deterministic whenever possible.

Randomness should enter only through carefully defined mechanisms, such as:

- mutations
- initial conditions
- environmental variation

This makes experiments reproducible.

---

## 9. Toy Before Science

This project is inspired by biology—not constrained by it.

If reality is unnecessarily complicated, choose the simpler rule.

Fun and understandability take priority over realism.

---

## 10. Nothing is Special

Everything in the universe—including organisms—obeys the same underlying laws.

Organisms are not privileged objects in the simulation but persistent structures that emerge from those laws.

---


# Design Philosophy

The simulation should resemble a sandbox more than a game.

There are:

- no objectives
- no scoring
- no levels
- no scripted events

The enjoyment comes from observing unexpected behaviors emerge from simple interactions.

---

# Scientific Inspiration

The project draws inspiration from:

- cellular automata
- artificial life
- evolutionary biology
- ecology
- neural systems
- physics
- information theory

These disciplines provide inspiration, but they are intentionally simplified into a coherent toy universe.

---

# Things We Explicitly Avoid

The project intentionally avoids:

- hard-coded intelligence
- scripted behaviors
- magical mechanics
- special-case organisms
- exceptions to physical laws
- optimization before understanding
- unnecessary realism

---

# Success Criteria

The project succeeds if observers repeatedly find themselves saying:

> "I didn't program that... why did it happen?"

Those moments of surprise are evidence that the underlying rules are producing genuine emergence.
